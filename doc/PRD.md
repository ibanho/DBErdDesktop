# DB ERD Desktop PRD

SSH 터널, 암호화 세션, DB 메타데이터 기반 ERD 및 문서화 도구

작성일: 2026-05-11

---

## 1. 제품 개요

DB ERD Desktop은 사용자가 MySQL, PostgreSQL, MSSQL, Oracle 데이터베이스에 연결하여 데이터베이스/스키마 아래의 테이블과 뷰를 확인하고, 선택한 테이블로 논리/물리 ERD를 생성한 뒤 문서로 저장하는 Python Qt6/QML 데스크톱 애플리케이션이다.

직접 DB 연결뿐 아니라 SSH 터널을 통한 보안 연결을 지원하며, 연결 정보와 계정 정보는 암호화된 세션 파일로 저장한다.

---

## 2. 문제 정의

- 운영/개발 DB가 사설망 내부에 있어 직접 접근이 어려움
- DB 구조를 문서화하려면 수동 캡처와 표 작성이 반복됨
- 테이블/컬럼 물리명이 영문 약어 위주라 비개발자에게 의미 전달이 어려움
- 연결 정보와 계정 정보를 매번 입력하는 과정이 번거로움
- 산출물 형식이 프로젝트마다 DOCX, HWPX, PPTX로 다름

---

## 3. 목표

- SSH 터널을 통해 사설망 DB에 안전하게 연결한다.
- DBMS별 메타데이터를 조회해 데이터베이스/스키마/테이블/뷰를 계층 트리 구조로 표시한다.
- 선택된 테이블의 관계를 ERD 연결선으로 시각화한다.
- 물리 ERD와 한글 의미 기반 논리 ERD를 모두 제공한다.
- 세션 정보를 암호화 저장하고 앱 시작 시 세션을 선택하거나 생성한다.
- QML로 UI/UX를 구성하고 Python 백엔드와 도메인 로직을 분리한다.
- DOCX, HWPX, PPTX 형식으로 문서화 결과를 저장한다.
- 앱은 싱글 인스턴스로 동작하며 중복 실행 시 기존 실행 창을 활성화한다.

---

## 4. 비목표

- DB 스키마 변경 기능은 제공하지 않는다.
- SQL 편집기 또는 데이터 조회 도구를 제공하지 않는다.
- 외부 AI API 기반의 의미 추론은 필수 기능으로 두지 않는다.
- 멀티 사용자 협업 저장소 또는 중앙 세션 서버를 제공하지 않는다.
- 웹 앱 또는 서버 기반 배포를 목표로 하지 않는다.

---

## 5. 대상 사용자

- 데이터베이스 설계 문서를 작성하는 개발자
- 운영 DB 구조를 파악해야 하는 유지보수 담당자
- 제안서, 산출물, 테스트 문서에 ERD를 포함해야 하는 프로젝트 담당자
- 사설망 DB에 SSH jump server를 통해 접근하는 엔지니어

---

## 6. 핵심 사용자 흐름

1. 앱을 실행한다.
2. 이미 실행 중인 인스턴스가 있으면 새 인스턴스는 종료되고 기존 창이 앞으로 이동한다.
3. 시작 모달 다이얼로그에서 기존 세션 열기 또는 새 세션 생성을 선택한다.
4. 새 세션을 선택한 경우 별도 세션 생성 화면에서 세션명과 설명을 입력한다.
5. SSH 서버 정보와 DB 계정 정보를 입력하거나 세션에서 불러온다.
6. SSH 터널 및 DB 연결을 수행한다.
7. 연결 후 DB 정보와 데이터베이스/스키마/테이블/뷰 트리를 확인한다.
8. 필요한 데이터베이스/스키마의 테이블을 선택한다.
9. 논리/물리 ERD를 생성한다.
10. 저장 메뉴에서 위치와 형식을 선택하여 저장한다.

---

## 7. 세션 요구사항

- 앱 첫 화면은 모달 다이얼로그로 제공한다.
- 사용자는 기존 세션 열기 또는 새 세션 생성을 선택해야 한다.
- 새 세션 생성은 시작 모달 내부 입력이 아니라 별도 세션 생성 화면에서 수행한다.
- 메인 메뉴의 `Session > Create Session`에서도 새 세션 생성 모달을 열 수 있어야 한다.
- DB 연결 중에는 새 세션 생성 메뉴를 비활성화해 의도하지 않은 연결 종료를 방지한다.
- 세션 생성 화면은 세션명과 선택 설명을 입력받고, 생성 후 연결정보 입력 화면으로 이동한다.
- 시작 모달에서 새 세션 생성 모달이 열리면 시작 모달은 입력을 받지 않는 비활성 상태가 되어야 한다.
- 세션을 연 이후의 메인 작업 화면에는 세션 선택/삭제/저장 패널을 노출하지 않는다.
- 세션을 연 이후의 메인 작업 화면에는 SSH/DB 계정 입력 폼을 노출하지 않는다.
- SSH/DB 연결정보 편집은 `Session > Edit Connection Settings...` 다이얼로그에서 수행한다.
- 현재 세션의 연결정보 저장은 메인 메뉴의 `Session > Save Current Session`으로 수행한다.
- 세션에는 SSH 연결 정보와 DB 연결 정보가 포함된다.
- 세션 파일은 암호화된 파일로 저장한다.
- 세션 파일 경로는 `%APPDATA%\DBErdDesktop\sessions.enc`를 기본값으로 한다.
- 세션 암호화 키는 `%APPDATA%\DBErdDesktop\session.key`에 저장한다.

## 8. UI/UX 구조 요구사항

- UI/UX는 QML 파일에서 정의한다.
- Python 코드는 세션, DB 연결, SSH 터널, ERD 생성, 문서 저장 로직을 담당한다.
- QML은 Python 백엔드의 프로퍼티와 슬롯을 통해 상태를 표시하고 사용자 액션을 전달한다.
- 화면 로직과 DB/문서 도메인 로직은 서로 직접 섞지 않는다.

---

## 9. SSH 터널 요구사항

- SSH 터널 사용 여부를 선택할 수 있어야 한다.
- SSH host, port, username, password를 입력할 수 있어야 한다.
- private key와 key passphrase를 입력할 수 있어야 한다.
- SSH 터널 연결 후 로컬 포트를 통해 DB에 접속해야 한다.
- 터널이 비활성화된 경우 DB에 직접 연결해야 한다.
- 연결 실패 시 사용자가 이해할 수 있는 오류 메시지를 표시해야 한다.

- Oracle

## 11. DB 정보 표시 요구사항

DB 연결 성공 후 다음 정보를 표시한다.

- DBMS
- 연결 방식: direct 또는 SSH tunnel
- 조회된 데이터베이스/스키마 수
- 조회된 테이블 수
- 조회된 뷰 수

---

## 12. 트리 선택 요구사항

- 데이터베이스 또는 스키마를 상위 노드로 표시한다.
- 데이터베이스 또는 스키마 아래에 `Tables` 그룹과 `Views` 그룹을 표시한다.
- 테이블과 뷰는 각 그룹의 하위 노드로 표시한다.
- 상위 노드 선택 시 하위 테이블 선택 상태가 함께 반영되어야 한다.
- 뷰 노드는 구조 확인용으로 표시하며 ERD 생성 선택 대상에는 포함하지 않는다.
- 여러 데이터베이스/스키마의 테이블을 선택할 수 있어야 한다.
- 선택된 테이블만 ERD 생성 대상이 된다.

---

## 13. ERD 요구사항

ERD는 두 가지 뷰를 제공한다.

- 논리 ERD
- 물리 ERD

공통 요구사항:

- 테이블은 박스 형태로 표시한다.
- 컬럼은 테이블 박스 내부에 표시한다.
- PK/FK 여부를 표시한다.
- FK 관계가 있는 테이블 사이에 연결선을 표시한다.
- 확대, 축소, 초기화 기능을 제공한다.

---

## 14. 논리 ERD 요구사항

- 테이블/컬럼 comment가 한글이면 comment를 우선 사용한다.
- comment가 없거나 한글이 아니면 물리명을 기반으로 한글 의미를 추론한다.
- 예시:
  - `user_id` → `사용자 식별자`
  - `created_at` → `생성 일시`
  - `email` → `이메일`
  - `order_no` → `주문 번호`
- 추론 결과는 ERD와 문서 산출물에 반영한다.

---

## 15. 물리 ERD 요구사항

- 테이블명은 실제 물리명을 표시한다.
- 컬럼명은 실제 물리명을 표시한다.
- 컬럼 타입을 표시한다.
- NULL/NOT NULL 여부를 표시한다.
- PK/FK 여부를 표시한다.
- 기본값이 있는 경우 문서 산출물에 포함한다.

---

## 16. 저장 요구사항

- `File > Save...` 메뉴를 제공한다.
- 저장 버튼도 제공한다.
- 저장 시 사용자가 저장 위치를 선택할 수 있어야 한다.
- 저장 시 문서 형식을 선택할 수 있어야 한다.

지원 형식:

- DOCX
- HWPX
- PPTX

---

## 17. 문서 산출물 요구사항

문서에는 다음 내용을 포함한다.

- 문서 제목
- DBMS
- 데이터베이스/스키마명
- 생성 일시
- 논리 ERD 이미지
- 물리 ERD 이미지
- 테이블별 컬럼 명세
- FK 관계 목록

DOCX/PPTX는 ERD PNG를 문서에 삽입한다.
HWPX는 문서 본문과 함께 ERD PNG 파일명을 기록한다.

---

## 18. 보안 요구사항

- 세션 파일에는 연결정보와 계정정보를 평문으로 저장하지 않는다.
- 세션 파일은 Fernet 기반 대칭키 암호화를 사용한다.
- 암호화 키 파일은 앱 데이터 폴더에 별도 저장한다.
- SSH password, DB password, private key passphrase는 UI에서 비밀번호 입력 필드로 표시한다.
- 연결 실패 시 비밀번호 값을 오류 메시지에 노출하지 않는다.

---

## 19. 패키징 요구사항

- Windows 단일 EXE로 배포한다.
- PyInstaller 기반 빌드 스크립트를 제공한다.
- 실행 파일 이름은 `DBErdDesktop.exe`로 한다.
- 빌드 결과는 `dist/DBErdDesktop.exe`에 생성한다.
- 재빌드 명령은 `build_exe.ps1`로 제공한다.

## 20. 실행 인스턴스 요구사항

- 앱은 사용자 환경에서 하나의 인스턴스만 실행되어야 한다.
- 첫 번째 인스턴스는 로컬 IPC 서버를 열어 실행 상태를 점유한다.
- 두 번째 실행 시도는 기존 인스턴스에 활성화 요청을 보낸 뒤 새 프로세스를 종료한다.
- 기존 인스턴스가 최소화되어 있으면 정상 창 상태로 복원한다.
- 기존 인스턴스에 시작 모달 등 활성 모달 창이 있으면 해당 모달 창을 앞으로 가져온다.

## 22. 수용 기준

- 앱 실행 시 세션 열기/생성 모달이 표시된다.
- 앱 UI는 QML에서 로드되고 Python 백엔드는 QML 프로퍼티/슬롯으로 연결된다.
- 앱이 이미 실행 중인 상태에서 다시 실행하면 두 번째 인스턴스는 생성되지 않는다.
- 중복 실행 시 기존 실행 창 또는 활성 모달 창이 앞으로 이동한다.
- `Session > Create Session` 메뉴를 누르면 새 세션 생성 모달이 표시된다.
- 새 세션 생성이 완료되면 해당 세션으로 전환되고 연결정보 모달이 표시된다.
- 시작 모달에서 새 세션 생성 모달을 열면 뒤쪽 시작 모달의 세션 선택/버튼 조작이 비활성화된다.
- 모달에서 기존 세션을 열면 메인 화면은 세션명과 작업 상태를 표시하고, 계정 입력 폼은 표시하지 않는다.
- 모달에서 새 세션 생성을 선택하면 별도 세션 생성 화면이 표시된다.
- 별도 세션 생성 화면에서 새 세션을 생성하면 암호화 세션 파일에 저장된다.
- 세션을 열고 메인 화면으로 이동하면 별도의 세션 설정 패널이 보이지 않는다.
- SSH 터널 사용 시 터널 연결 후 DB 연결이 수행된다.
- 연결 후 DB 정보가 화면에 출력된다.
- 데이터베이스/스키마 아래에 테이블과 뷰가 그룹별 계층 트리 구조로 표시된다.
- 선택한 테이블 기준으로 논리/물리 ERD가 생성된다.
- FK 관계가 연결선으로 표시된다.
- DOCX/HWPX/PPTX 저장이 가능하다.
- EXE 실행 파일로 앱을 실행할 수 있다.

- 세션별 마지막 선택 테이블 저장
